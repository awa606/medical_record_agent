# ASR 遗留问题与修复计划

## 当前问题

- 转写段落过碎，单独的“嗯、呢、哦”等口头语会占据多行，影响医生快速阅读。
- 语句边界不自然，同一说话人的连续短句没有合并成自然问答。
- 角色显示仍需稳定：声学说话人和医生/患者角色必须分开处理，不能把低置信度逐句猜测当作事实。
- 单人朗读医患脚本时，系统不能伪造成真实医生和患者两个人。

## v0.9.1 修复范围

- 恢复 `speaker_diarization.py` 中医生、患者、其他、待确认等中文规则文本。
- 合并短口头语声纹簇，优先并入相邻稳定说话人。
- 合并同一说话人短间隔 utterance，减少碎片化转写行。
- 保留医生问句和患者回答的边界，不跨 speaker 合并。
- 单 speaker 音频只标记为一位说话人并进入全局确认，不伪造医患双人。
- 前端转写区轻量清理：保留时间、角色、正文和校正入口，将重复的“查看原文”改为“详情”。

## 不在本轮处理

- 不实现浏览器麦克风实时录音。
- 不承诺自动角色识别 100% 正确。
- 不继续重做整体三栏 UI。
- 不把临床角色低置信度判断直接写成最终事实。

## 验收标准

- `fever_01.wav` 等课程样本不应大量出现独立“嗯”“呢”行。
- 医生问句和患者回答不应合并到同一稳定 utterance。
- 同一 speaker 完成全局角色映射后，所有发言同步更新。
- 单人朗读样本不伪造成医生/患者两人。
- 病历预览只消费稳定转写段，避免临时 streaming 文本污染主诉、现病史和证据定位。

## 后续计划

- `v0.9.3`：整理最终演示闭环、交付检查清单和评审材料。
