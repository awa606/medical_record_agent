# 开发日志机制

本目录用于记录 Medical Record Agent 的关键开发变更。后续每次代码、接口、前端、测试或文档结构发生实质修改时，都必须新增或更新对应开发日志。

## 使用规则

1. 新功能、新页面、新接口、新测试：新增一篇版本或任务日志。
2. 修复已有功能：更新对应版本日志，或新增修复日志。
3. 只改文案、课程说明或验收说明：仍需记录修改目标、文件和验证方式。
4. 不提交真实患者数据、真实 API Key、模型权重或大体积数据集文件。
5. 每篇日志必须覆盖 `TEMPLATE.md` 中列出的字段。
6. 具体开发规范见 `DEVELOPMENT_RULES.md`。

## 命名建议

- 版本日志：`V0.4_doctor_debug_index_split.md`
- 修复日志：`V0.4.4_fix_doctor_api_flow.md`
- 文档日志：`V0.4.5_dev_log_mechanism.md`
