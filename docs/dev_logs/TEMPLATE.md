# 开发日志模板

## 修改日期 / 时间

YYYY-MM-DD HH:mm，时区：Asia/Shanghai

## 修改目标

说明本次修改要解决什么目标，属于哪个版本或任务。

## 修改前问题

说明修改前的缺陷、缺口、限制或课程验收风险。

## 输入

列出需求来源、用户指令、样例数据、接口、测试样例或参考文档。

## 输出

列出本次产出的功能、页面、接口行为、文档或测试结果。

## 修改文件

- `path/to/file`

## 关键设计决策

记录为什么这样设计，以及放弃了哪些方案。

## 验证步骤

1. 运行命令或打开页面。
2. 输入样例。
3. 检查输出。

## 验证结果

说明命令结果、页面现象、测试是否通过。

## 未解决问题

列出仍未完成、受环境限制或需要下一阶段处理的问题。

## 下一步计划

列出下一阶段应做的事项。
