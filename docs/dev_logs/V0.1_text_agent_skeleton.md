# V0.1 文本版 Agent 骨架

## 修改时间

回顾补写：2026-06-08，时区：Asia/Shanghai

## 修改目标

建立基于医患对话文本的门诊病历生成 Agent POC，验证“输入问诊文本 -> 抽取结构化字段 -> 生成病历草稿 -> 安全校验 -> 等待医生审核”的主链路。

## 修改前问题

项目还没有可运行的病历生成主流程，也没有任务状态、步骤记录和医生审核边界。课程展示只能停留在普通脚本或静态表单，无法体现智能体流程。

## 输入

- 模拟医患问诊文本。
- Mock LLM 规则。
- 门诊病历字段需求：主诉、现病史、既往处理、伴随症状、既往史、过敏史、查体、候选诊断。

## 输出

- 文本生成病历接口。
- Agent 任务状态流转。
- 字段抽取、草稿生成、安全校验和医生审核等待状态。
- SSE 任务进度基础能力。

## 关键设计决策

- 使用 Mock LLM 而不接真实 LLM，保证课程项目可离线、可复现、无隐私风险。
- 任务结束停在 `WAITING_DOCTOR_REVIEW`，明确 AI 只生成草稿，不自动替代医生确认。
- 使用任务表、步骤表和审计日志保留 Agent 过程证据，便于课程答辩解释智能体链路。

## 验证步骤

1. 启动后端服务。
2. 调用 `POST /api/records/generate` 提交模拟问诊文本。
3. 监听 `/api/tasks/{task_id}/events`。
4. 查询 `/api/tasks/{task_id}` 和 `/api/tasks/{task_id}/steps`。
5. 运行单元测试。

## 验证结果

任务可从创建进入字段抽取、草稿生成、安全校验，并最终停在医生审核阶段。测试覆盖主流程、任务状态和安全校验。

## 未解决问题

- 仅支持文本输入。
- 字段抽取依赖规则和 Mock LLM，不具备真实医学泛化能力。
- 没有音频输入和 ASR 评测。

## 下一步

新增音频入口和 Mock ASR，把预录音频转写结果接入既有文本主流程。
