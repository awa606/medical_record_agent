# V0.2 Mock ASR 与三入口

## 修改时间

回顾补写：2026-06-08，时区：Asia/Shanghai

## 修改目标

在 V0.1 文本病历生成链路基础上，新增三入口：从文本生成病历、上传预录音频测试转写、上传预录音频生成病历。

## 修改前问题

系统只能处理文本，无法演示音频进入病历 Agent 的链路；前端也更像单一调试表单，缺少课程展示所需的多入口流程。

## 输入

- 任意 `.wav` 或 `.mp3` 预录音频。
- Mock ASR 固定蛇咬伤问诊样例。
- V0.1 文本病历生成接口。

## 输出

- `ASRResult`、`ASRSegment`、`AudioRecord` schema。
- `/api/audio/upload`
- `/api/audio/{audio_id}/transcribe`
- `/api/audio/{audio_id}/transcript`
- `/api/audio/{audio_id}/generate-record`
- 三入口前端页面。

## 关键设计决策

- Mock ASR 只用于工程链路验证，不声称真实转写能力。
- ASR 输出统一成 `ASRResult`，避免后续接 FunASR、Online ASR 或 Qwen3 时改病历 Agent 主流程。
- 音频生成病历复用 V0.1 的 `conversation_text -> records/generate` 链路，保持边界清晰。

## 验证步骤

1. 上传音频文件。
2. 调用 Mock ASR 转写。
3. 检查 `ASRResult.text`、`conversation_text`、`segments`、`medical_keywords`。
4. 调用音频生成病历。
5. 运行 `python -m pytest`。

## 验证结果

Mock ASR 可返回固定医患对话，音频生成病历可复用原 Agent 主链路。相关 API 和测试通过。
