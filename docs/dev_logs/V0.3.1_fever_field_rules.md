# V0.3.1 fever 字段抽取规则

## 修改时间

回顾补写：2026-06-08，时区：Asia/Shanghai

## 修改目标

补强 fever 病例的字段抽取规则，使 `fever_01` 能从问诊文本中抽取主诉、现病史、既往处理、伴随症状和候选诊断，并生成非空病历草稿。

## 修改前问题

FunASR 已能完成 fever 样例转写，关键词缺失可做到 `missing=[]`，但病历字段抽取规则仍偏向早期蛇咬伤样例，fever 病例结构化输出不够稳定。

## 输入

- fever clean 文本。
- `fever_01.wav` 转写结果。
- 关键词：发热、40 度、咳嗽、铁锈色痰、布洛芬。

## 输出

- fever 病例规则识别。
- fever 字段抽取。
- fever 候选诊断。
- fever 草稿生成测试。

## 修改文件

- `app/services/mock_llm.py`
- `tests/test_mock_llm_fever.py`
- `tests/test_prompts_and_mock_llm.py`
- `docs/asr_v0_3.md`

## 关键设计决策

- 仍使用 Mock LLM 规则增强，不接真实 LLM。
- 候选诊断只标为“候选/待医生确认”，不自动给最终诊断。
- 查体、过敏史等未在对话中出现的字段保持待补充，体现医生审核必要性。

## 验证步骤

1. 使用 fever clean 文本调用文本生成病历。
2. 检查字段值、缺失项和候选诊断。
3. 检查草稿不为空。
4. 运行 `python -m pytest tests/test_mock_llm_fever.py`。

## 验证结果

fever 病例可稳定生成结构化字段和病历草稿，候选诊断仍需医生确认，缺失字段会被安全校验提示。

## 未解决问题

- 规则只覆盖课程样例，不具备真实临床泛化能力。
- 查体和处理建议需要医生补充。
- ASR 单段角色校正仍未自动解决。

## 下一步

优化前端医生工作台，使字段、转写、证据和安全校验更接近真实电子病历工作流。
