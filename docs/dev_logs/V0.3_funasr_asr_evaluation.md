# V0.3 FunASR 接入与 ASR 评测

## 修改时间

回顾补写：2026-06-08，时区：Asia/Shanghai

## 修改目标

接入 FunASR 批量转写能力，并建立 ASR 评测指标，用 CER 和医疗关键词召回率对比不同 ASR 引擎。

## 修改前问题

V0.2 只有 Mock ASR，不能证明真实音频能转写，也不能量化 ASR 准确率。缺少对 `fever_01.wav` 等样例的客观验收方式。

## 输入

- `fever_01.wav`、`snakebite_01.wav` 等预录音频。
- FunASR 可选依赖。
- ASR manifest、热词和人工标注文本。

## 输出

- `engine=funasr`。
- `ASREvaluator`，输出 CER、keyword_recall、recognized、missing。
- `/api/audio/{audio_id}/evaluate`
- ASR 文档 `docs/asr_v0_3.md`。
- 前端 ASR 评测入口。

- `app/services/asr/funasr_engine.py`

- `app/services/asr/factory.py`
- `app/services/asr/role_strategy.py`

- `tests/test_asr_role_strategy.py`

## 关键设计决策

- FunASR 作为可选依赖，不放入基础 `requirements.txt`，避免未安装真实 ASR 时影响文本主流程。
- 评测聚焦 CER 和关键词召回，不把医生/患者角色分离作为 V0.3 的硬保证。
- 当真实 ASR 返回单段长文本时，使用 `single_segment_needs_review` 提醒人工校正，避免错误强行标注角色。

## 验证步骤

1. 安装 ASR 可选依赖。
2. 运行 `python scripts/check_funasr_env.py`。
3. 上传 `fever_01.wav`。
4. 调用 `engine=funasr` 转写。
5. 调用 `/api/audio/{audio_id}/evaluate`。
6. 检查 CER、keyword_recall、recognized、missing。

## 验证结果

在具备 FunASR 环境时，`fever_01.wav` 可完成真实转写；关键词评测可用于和 Mock、Online、Qwen3 做对比。未安装 FunASR 时，接口返回清晰错误，不影响其他功能。

## 未解决问题

- FunASR 首次运行依赖模型下载和本地环境。
- 单段长文本仍需要人工校正医生/患者角色。
- 未做实时 ASR。
