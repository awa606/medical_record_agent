# V0.4.3 医生工作台 UI 优化

## 修改时间

回顾补写：2026-06-08，时区：Asia/Shanghai

## 修改目标

把医生端首屏从入口调试页调整为真实医生工作台：顶部信息、流程步骤、左中右三栏、右栏 AI 辅助与安全校验、隐藏调试详情。

## 修改前问题

页面一打开仍像调试页，大文本框、入口卡片和 JSON 信息过于显眼，医生审核所需的字段、转写、证据、安全校验没有形成稳定工作台布局。

## 输入

- Issue #27 的多视图医生工作台要求。
- 医疗工作台参考图。
- fever 病例和 FunASR 单段角色校正提醒需求。

## 输出

- 默认三栏工作台。
- 顶部小工具栏。
- 字段卡片状态 badge。
- 医患对话样式。
- 右栏缺失项、候选诊断、字段证据、安全校验、ASR 评测摘要。
- 调试 JSON 默认隐藏。

## 修改文件

- `static/index.html`
- `static/style.css`
- `static/main.js`
- Figma 原型：`V0.4.3 默认医生工作台 - 三栏首屏`

## 关键设计决策

- 首屏直接显示医生工作台，不再先显示入口选择页。
- 文本导入、ASR 评测和调试详情使用抽屉，避免长期占用主界面。
- `single_segment_needs_review` 在中栏和右栏同时提示，确保医生不会忽略角色校正风险。
- 使用白底、浅灰背景、低饱和蓝色和少量红黄绿状态色，贴近医疗软件而非营销页。

## 验证步骤

1. 打开 `/static/index.html`。
2. 检查默认三栏、五个工具按钮和空状态。
3. 点击文本导入，提交 fever 文本。
4. 等待 SSE 回填字段、草稿和安全校验。
5. 打开调试抽屉，检查 JSON 不在主界面。

## 验证结果

页面首屏三栏布局成立，文本导入真实链路可回填，主界面不显示 JSON。Figma 原型同步生成。

## 未解决问题

- 职责仍集中在 `index.html`，医生端和调试端没有完全分离。
- 后续需要拆分成 `doctor.html`、`debug.html` 和入口页。

## 下一步

进行 V0.4 页面职责拆分，并在第二阶段将医生端接入真实 API。
