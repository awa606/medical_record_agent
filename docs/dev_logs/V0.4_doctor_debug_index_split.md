# V0.4 doctor/debug/index 三页面拆分

## 修改时间

回顾补写：2026-06-08，时区：Asia/Shanghai

## 修改目标

拆分前端职责：`index.html` 作为入口页，`debug.html` 保留开发调试功能，`doctor.html` 作为医生端工作台页面。

## 修改前问题

所有功能集中在 `index.html`，页面既承担调试、上传、评测、JSON 查看，又试图作为医生工作台，导致首屏像开发调试页，不像真实医生端系统。

## 输入

- 现有 V0.4.3 工作台页面。
- 保留三入口和 ASR 评测的要求。
- 医生端先静态、后接 API 的分阶段要求。

## 输出

- `static/index.html`：入口页。
- `static/debug.html`：开发调试页。
- `static/doctor.html`：医生端页面。
- `static/doctor.css`：医生端独立样式。
- `static/doctor.js`：医生端逻辑。

## 修改文件

- `static/index.html`
- `static/debug.html`
- `static/doctor.html`
- `static/doctor.css`
- `static/doctor.js`
- `static/style.css`
- `static/main.js`

## 关键设计决策

- 不删除调试能力，而是迁移到 `debug.html`，降低医生端噪声。
- 医生端不直接展示大段 JSON，JSON 放入调试抽屉或调试页。
- 初版医生端先用 fever 静态数据确保视觉和信息架构通过，再接入真实接口。

## 验证步骤

1. 打开 `/static/index.html`。
2. 检查只显示“进入医生端”和“进入调试页”。
3. 打开 `/static/doctor.html`，检查三栏工作台。
4. 打开 `/static/debug.html`，检查原 ASRResult、Task、Steps、Safety JSON 调试仍存在。
5. 运行 `node --check static/doctor.js` 和 `python -m pytest`。

## 验证结果

三页面职责分离完成，医生端主页面不再承担调试功能，调试页保留完整开发入口。

## 未解决问题

- 初版 doctor 页只使用静态数据。
- 医生端尚未接入真实后端接口。

## 下一步

将 `doctor.js` 接入 `/api/records`、`/api/audio` 和 `/api/tasks`，使医生端可真实生成病历。
