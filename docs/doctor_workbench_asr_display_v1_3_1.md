# v1.3.1 医生端 ASR 角色展示验收记录

## 目标

本轮只修复 ASR 转写区的角色展示可用性，不调整整体三栏布局，不修改后端业务 API。

核心规则：

- 低置信度或未人工确认的自动角色，不再直接显示为“医生/患者”。
- 有声学说话人身份时，默认显示“说话人 A/B/C”。
- 医生在详情抽屉中完成整位说话人映射后，同一 speaker 的全部发言统一更新为“医生/患者/其他”。
- 病历预览和正式生成继续依赖稳定转写段，不能把未确认角色伪装成事实。

## 验收结论

- 默认医生界面不再把低置信度自动判断包装成确定事实。
- 不再逐句弹出大量“待确认”，而是保留一次全局 speaker-role 映射。
- 转写区仍保留时间、角色、正文、详情入口，原有播放器、证据定位和角色校正流程不受影响。

## 已知边界

- 本轮没有提升 ASR 模型本身的文字准确率。
- 本轮没有实现浏览器麦克风真流式录音。
- 对真实多说话人音频，自动 speaker 聚类仍可能出错；系统只保证低置信度时不冒充“医生/患者”确定结论。
