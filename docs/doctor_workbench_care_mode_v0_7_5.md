# v0.7.5 医生端关怀版布局验收记录

## 本次完成

- 新增 `标准版 / 关怀版` 显示模式切换。
- 关怀版通过 `body.care-mode` 样式放大字号、按钮、行距和状态标识。
- 关怀版继续复用标准版接口、状态和按钮事件，不复制后端逻辑。
- 关怀版默认减少转写摘要数量，优先展示当前步骤、关键字段、主要转写、AI 摘要和操作按钮。

## 前端展示调整

- 顶部新增 `标准版 / 关怀版` 切换控件。
- 关怀版字体和按钮明显变大，适合演示、投屏和长时间使用。
- 右侧操作区不重叠，确认字段、保存草稿和确认导出流程不变。

## 怎么测试

```powershell
docker compose up -d --build
curl http://127.0.0.1:2601/health
node --check static\doctor.js
```

前端手工测试：

1. 打开 `http://127.0.0.1:2601/static/doctor.html`。
2. 点击顶部 `关怀版`。
3. 检查页面字号、按钮和行距是否明显变大。
4. 点击 `输入方式 -> 文本生成`，生成一条病历。
5. 检查字段区、转写区、AI 摘要区和操作区不重叠。
6. 点击详情按钮，确认抽屉正常打开。
7. 点击 `标准版`，确认能切回标准布局。

## 实际验收结果

- `node --check static\doctor.js`：通过。
- `docker compose up -d --build`：通过。
- 浏览器验收：关怀版切换成功，无横向溢出，右侧操作区不重叠，详情抽屉正常打开。
- 截图证据：`docs/final_report/images/v0_7_5_care_layout/`。

## 说明

关怀版为大字号、高可读性布局，允许出现必要的内容滚动；标准版继续承担一屏化和低滚动的主要展示目标。
