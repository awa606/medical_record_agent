# v0.7.8 病历草稿固定槽位与 AI 三块区验收说明

## 本次完成

- 顶部信息栏保持不动，只调整流程条以下医生端工作台。
- 左栏改为 `病历草稿`，固定展示主诉、现病史、既往史、过敏史、查体、初步诊断、处理建议。
- 生成前也保留七个空白槽位，避免单个大空态块和大片空白。
- `初步诊断` 从候选诊断摘要生成，不修改后端数据结构。
- 右栏 `AI辅助` 只保留候选诊断、治疗方案、诊断证据三块。
- 安全校验从右侧主卡片隐藏，但仍用于导出阻断和医生确认边界。
- 浏览器 100% 缩放下，通过工作台局部字号和间距调整，接近用户在 75% 缩放时认可的空间分布。

## 前端展示变化

- 病历草稿字段槽位先占位，内容为空时仍能看到完整病历结构。
- 字段内容自动换行，卡片高度保持稳定，内容过长时在卡片内部滚动。
- 每个字段仍保留 `证据` 和 `详情` 入口，便于查看完整内容。
- 右侧不再堆叠缺失提醒和安全校验结果，减少视觉噪声。
- 操作区继续固定在右栏底部，和 AI 内容不重叠。

## 怎么测试

```powershell
node --check static\doctor.js
$env:PYTHONPATH=(Get-Location).Path
pytest -q
git diff --check
docker compose up -d --build
curl http://127.0.0.1:2601/health
curl -I http://127.0.0.1:2601/static/doctor.html
```

前端手工验收：

1. 打开 `http://127.0.0.1:2601/static/doctor.html`，浏览器缩放保持 100%。
2. 空态下确认左栏显示七个病历草稿槽位。
3. 确认顶部信息栏、ASR Engine、输入方式按钮没有变化。
4. 使用 `输入方式 -> 文本生成` 生成病历。
5. 检查左栏字段自动换行，长内容可局部滚动或通过详情小窗查看。
6. 检查右栏只显示候选诊断、治疗方案、诊断证据三块。
7. 检查操作区固定在右栏底部，不遮挡 AI 内容。
8. 上传 Mock ASR 音频，确认 SSE 转写、角色校正、病历生成、审核和导出流程不受影响。
