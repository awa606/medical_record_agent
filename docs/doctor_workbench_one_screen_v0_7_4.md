# v0.7.4 标准医生端一屏化布局验收记录

## 本次完成

- 压缩顶部状态栏、流程条和下一步提示区，减少首屏上方占用。
- 标准医生端改为摘要优先：左侧只显示 4 个关键病历字段，中间默认显示 4 条转写摘要，右侧显示 4 个 AI 摘要卡。
- 新增统一详情抽屉：完整病历字段、候选诊断、转写全文、治疗建议、判断证据和安全校验均通过“详情”查看。
- 操作区保持在右侧底部，按钮 ID 和原有流程不变。

## 前端展示调整

- 首屏布局更接近“顶部状态 + 流程 + 三栏工作台”，下方不再留大片空白。
- 标准版生成后不出现明显的多重滑块；长内容不堆在首屏。
- AI 辅助区保留候选诊断、治疗方案推荐、判断证据、安全校验结果四块，但默认只展示关键摘要。

## 怎么测试

```powershell
docker compose up -d --build
curl http://127.0.0.1:2601/health
curl -I http://127.0.0.1:2601/static/doctor.html
node --check static\doctor.js
```

前端手工测试：

1. 打开 `http://127.0.0.1:2601/static/doctor.html`。
2. 点击 `输入方式 -> 文本生成`，粘贴蛇咬伤问诊样例并生成病历。
3. 检查左侧只显示关键字段摘要，中间只显示少量转写摘要，右侧显示四个 AI 摘要卡。
4. 点击 `查看全部字段、证据和候选诊断`，确认详情抽屉打开。
5. 检查页面无横向溢出，AI 区和操作区不重叠。

## 实际验收结果

- `node --check static\doctor.js`：通过。
- `docker compose up -d --build`：通过。
