# v1.3.6 录音生成入口保留与自动分类方向验收

## 本轮目标

- 保留医生端 `录音生成` 入口，避免把未来浏览器麦克风能力从产品路径中删除。
- 将 `Mock 演示` 独立为评审保底入口。
- 明确角色分类默认由程序自动完成，不要求医生逐句分类。

## 前端调整

`开始生成` 菜单现在包含：

- `录音生成`：保留接口，当前提示后续接入浏览器麦克风。
- `上传音频`：上传 MP3/WAV 生成病历。
- `粘贴文本`：文本输入生成病历。
- `Mock 演示`：自动切换到 mock ASR，作为现场评审保底流程。

## 自动分类口径

- 默认程序自动完成说话人和角色分类。
- 医生不逐句分类。
- 低置信度结果通过质量门禁、全局映射或后台提示兜底，不直接污染正式病历。
- 真实麦克风录音接入放到后续版本，不阻塞本周评审。
