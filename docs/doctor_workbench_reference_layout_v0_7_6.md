# v0.7.6 医生端参考图式布局修复说明

## 本次目标

本轮修复 `v0.7.4` 和 `v0.7.5` 过度压缩后导致的医生端不直观问题。标准医生端重新回到参考图式医疗工作台：顶部信息清晰、流程条清晰、三栏比例稳定、模块有合理占位；没有内容时保留空态，有内容时优先直接展示核心信息，超过区域再在对应模块内部滚动。

## 前端展示调整

- 顶部信息栏保留患者、会话、录音状态、ASR Engine 和审核状态，但字号和高度保持可读，不再过度挤压。
- 左侧病历字段恢复完整卡片展示：主诉、现病史、既往史、过敏史、查体、候选诊断、处理建议。
- 中间对话转写恢复逐行展示：时间、医生/患者角色、完整文本，内容较多时只在转写区内部滚动。
- 右侧固定四块：候选诊断、治疗方案推荐、判断证据、安全校验结果，底部保留操作区。
- 空态模块保留合理高度，不把页面撑得空旷，也不压缩到看不清。
- 关怀版继续保留入口，但复用同一布局体系放大字号和按钮，避免右侧拥挤和重叠。

## 修改文件

- `static/doctor.css`：增加 `v0.7.6 reference-style balanced doctor workbench` 覆盖规则，恢复均衡三栏和局部滚动。
- `static/doctor.js`：标准医生端不再只显示摘要，病历字段、转写行、AI 辅助卡片优先展示完整内容。
- `static/doctor.html`：更新静态资源版本号，避免浏览器使用旧缓存。

## 怎么测试前端

1. 打开 `http://127.0.0.1:2601/static/doctor.html`。
2. 空态下检查三栏是否都有合理占位，不出现大片无意义空白或拥挤卡片。
3. 点击 `输入方式 -> 文本生成`，生成一条病历。
4. 检查左侧病历字段、中间对话转写、右侧 AI 辅助和安全校验是否一眼可读。
5. 输入长文本或生成长内容时，确认只在病历字段区、转写区或 AI 内容区内部滚动。
6. 切换 `关怀版`，确认字号和按钮变大，但布局不重叠。
7. 上传 Mock ASR 音频，确认 SSE、角色校正、生成病历、审核导出流程不受影响。
