# v0.7.7 参考图式医生工作台精准修复说明

## 本次目标

本轮只修复流程条以下的医生端工作区，不再调整最上方信息栏。目标是让页面更接近参考图：横向流程清楚，三栏工作区均衡，病历字段、对话转写、AI 辅助和操作区都有稳定占位。

## 前端展示调整

- 顶部信息栏保持现状，不新增 `.topbar`、品牌区、患者信息、ASR Engine 或输入方式相关样式。
- 流程条改为更清晰的居中进度线，减少拥挤感。
- 下一步提示区改为轻量横条，只保留当前关键动作和右侧主操作按钮。
- 左侧病历字段卡片恢复参考图式卡片层级：标题、正文、状态标签、证据和详情入口。
- 中间对话转写恢复表格式阅读：时间、角色、文本，高亮当前对话行。
- 右侧 AI 区标准版改为四块等分：候选诊断、治疗方案推荐、判断证据、安全校验结果；正文只显示短摘要，详情通过按钮查看。
- 操作区固定在右栏底部，和 AI 内容保持间距，不重叠。

## 修改范围

- `static/doctor.css`：新增 `v0.7.7 reference workbench repair` 样式段，只覆盖流程条、下一步提示、三栏工作区、卡片和操作区。
- `static/doctor.js`：字段正文不再被 86 字硬截断，改由左栏内部滚动承载长内容。
- `static/doctor.html`：仅更新静态资源版本号，避免浏览器缓存旧样式。

## 怎么测试

1. 打开 `http://127.0.0.1:2601/static/doctor.html`。
2. 确认最上方信息栏没有变化。
3. 空态下检查三栏占位是否均衡。
4. 点击 `输入方式 -> 文本生成`，生成病历。
5. 确认右侧能同时看到四个 AI 模块标题和操作区。
6. 滚动左侧字段列表，确认只有字段列表内部滚动。
7. 切换 `关怀版`，确认无横向溢出、右侧操作区不重叠。
8. 上传 Mock ASR 音频，确认 SSE、角色校正、生成、审核、导出流程不受影响。
