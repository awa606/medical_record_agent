# v0.7.0 医生端视觉产品化记录

本轮目标是保持现有功能不变，将医生端页面调整为更接近医疗工作台的三栏布局：顶部状态栏、横向流程、左侧病历字段、中间对话转写、右侧 AI 辅助与安全校验。

## 本次前端展示调整

- 顶部信息栏：将患者、会话、录音状态、ASR 引擎和审核状态合并进顶部栏，减少首屏占用高度。
- 流程条：将原按钮式步骤改为圆点进度条，突出当前阶段和已完成阶段。
- 三栏工作区：保留“病历字段 / 对话转写 / AI辅助与安全校验”结构，调整列宽、间距、边框和卡片密度。
- 转写区：将对话段落改为更接近表格式的时间、角色、文本布局，保留角色校正和文本编辑能力。
- 操作区：保留底部重新生成、保存草稿、确认字段、确认导出按钮，视觉上压缩为更轻的操作条。
- 响应式保护：窄屏下顶部栏自动换行，避免横向溢出。

## 怎么在前端测试

1. 启动 Docker：`docker compose up -d --build`
2. 打开页面：`http://127.0.0.1:2601/static/doctor.html`
3. 点击“文本导入”，使用默认问诊文本生成病历。
4. 检查首屏是否完整显示顶部状态栏、流程条、三栏工作区和底部操作条。
5. 检查生成态是否显示病历字段、对话转写、候选诊断和安全校验。
6. 上传短 MP3/WAV，选择 Mock ASR 或真实 ASR，确认中间栏仍能显示 SSE 分段和角色校正。
7. 上传 16/30 分钟长音频时，确认切片状态仍显示在对话转写状态面板中。

## 验证结果

- `node --check static\doctor.js`：通过。
- `docker compose up -d --build`：通过。
- `curl http://127.0.0.1:2601/health`：返回 `{"status":"ok"}`。
- 浏览器验收：默认 1280 宽度无横向溢出，2048×1152 宽屏下三栏工作区完整显示。

## 边界说明

- 本轮只改前端展示层，不新增后端 API。
- 不修改 `ASRResult`、SSE 事件协议、病历生成、医生审核和导出流程。
- 医生模式隐藏内部调试/截图模式开关，调试入口仍保留在代码中，后续可按需重新暴露。
