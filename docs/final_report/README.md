# final_report 目录说明

本目录用于存放期末报告正式版材料。这里的文件只服务课程提交和答辩，不包含真实患者数据、真实 API Key、数据库文件、模型缓存或大体积音视频。

## 文件用途

| 文件 | 用途 |
| --- | --- |
| `AI生成式电子病历辅助系统_期末报告_正式版.md` | 期末报告正式版 Markdown，包含封面、摘要、关键词、目录、正文 12 章、提交说明和附录 |
| `截图清单.md` | 指导报告中需要插入哪些截图、每张图放在哪一章、截图目的和说明文字 |
| `报告提交检查清单.md` | 提交前人工验收清单，覆盖报告结构、评分点、截图证据和提交安全 |
| `README.md` | 本目录说明和下一步转 Word 指引 |

## 推荐工作流

1. 先阅读 `AI生成式电子病历辅助系统_期末报告_正式版.md`，补齐封面信息中的姓名、学号、指导老师、学院 / 专业等占位内容。
2. 按 `截图清单.md` 采集 10 张关键截图，并插入报告对应章节。
3. 根据课程格式要求调整标题、页眉页脚、页码和图片编号。
4. 使用 `报告提交检查清单.md` 做最终核对。
5. 确认没有真实 API Key、真实患者数据、SQLite 数据库、大体积音视频或模型缓存。

## Markdown 转 Word

如果当前环境已经安装 `python-docx`，可以运行：

```powershell
python scripts\export_final_report_docx.py
```

输出文件：

```text
docs/final_report/AI生成式电子病历辅助系统_期末报告_正式版.docx
```

如果提示缺少 `python-docx`，先安装：

```powershell
python -m pip install python-docx
```

也可以使用 VS Code、Typora、Pandoc 或其他 Markdown 工具手动导出 Word。导出后建议人工检查：

- 封面是否需要单独成页。
- 目录是否需要使用 Word 自动目录。
- 图片是否已经插入并带编号。
- 表格是否换页合理。
- 字体、行距、页边距是否符合课程要求。

## 提交注意

提交 GitHub 或课程平台前，只提交报告材料和必要脚本，不提交：

- `.venv/`
- `data/uploads/`
- SQLite 数据库文件
- `video/`
- 模型缓存
- 真实 API Key
- 真实患者数据
- 其他课程目录
