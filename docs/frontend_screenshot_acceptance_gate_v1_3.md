# 医生端截图验收门禁（v1.3 起执行）

从 v1.3 开始，所有涉及医生端前端、输入流程、转写流程、病历生成流程的迭代，都必须完成截图验收后才能提交。

## 必须截图的状态

- 空态首页：未上传、未导入文本。
- 输入方式：打开输入方式选择或输入弹窗。
- 转写中：至少包含进度、转写列表或等待提示。
- 生成后：左侧病历草稿、中间转写、右侧 AI 辅助均有内容。
- 详情抽屉：字段详情、证据、质量判断或导出阻断原因。
- 导出阻断或导出成功：至少覆盖一种导出状态。

## 必须覆盖的视口

- 1366x768，浏览器 100%。
- 1920x1080，浏览器 100%。

## 自查清单

每张截图都要记录：

- 是否存在遮挡或重叠。
- 是否出现横向溢出或不必要的页面级滚动。
- 信息是否过多，医生是否能一眼看到下一步。
- 主按钮是否清楚可见。
- 详情是否进入弹窗/抽屉，而不是堆在默认界面。
- 本次迭代目标是否达到。

## 证据保存路径

截图按版本保存到：

```text
docs/final_report/images/v1_3_xxx/
```

验收结论写入当天 daily log 和 `docs/能力证据追踪矩阵.md`。

## v1.3.0 本轮重点

- 流程条第 5、6 步不遮挡。
- 详情抽屉默认不覆盖整个工作区。
- AI 辅助区内容多时只在卡片内部滚动，操作区始终可见。
- 查体字段不再把医生追问、患者既往史、家族史误写入查体。
