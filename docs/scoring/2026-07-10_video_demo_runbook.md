# 2026-07-10 周评审视频展示 Runbook

本文档用于明天录屏和现场审核。目标是先保证完整业务闭环稳定可展示，再展示实时转写能力，不把评审风险压在本地 ASR 模型首次加载上。

## 1. 演示前检查

在项目根目录执行：

```powershell
docker compose up -d --build medical-record-agent
docker compose ps
Invoke-RestMethod http://127.0.0.1:2601/health
```

预期结果：

- `medical-record-agent` 状态为 `running` 和 `healthy`。
- `http://127.0.0.1:2601/health` 返回 `{"status":"ok"}`。
- 浏览器打开 `http://127.0.0.1:2601/static/doctor.html`。

## 2. 推荐录屏顺序

### 2.1 文本生成闭环，作为主线

1. 打开医生端 `http://127.0.0.1:2601/static/doctor.html`。
2. 选择文本输入或文本生成路线。
3. 展示左栏病历草稿：主诉、现病史、既往史、过敏史、查体、初步诊断、处理建议。
4. 展示右栏 AI 辅助：缺失项、候选诊断、证据、安全校验、操作区。
5. 展示确认字段、保存草稿、确认导出的医生审核边界。

讲解重点：

- 系统不是自动诊断，而是医生审核辅助。
- AI 只生成草稿和候选提示，最终导出必须医生确认。

### 2.2 Mock ASR 实时转写，展示 SSE 分段

1. 切换 ASR 引擎为 `mock`。
2. 上传项目内课程样例音频，不使用真实患者音频。
3. 观察中栏对话转写逐行追加。
4. 点击任一转写行或“显示设置”，打开“对话转写与校正”抽屉。
5. 修改角色或文本后保存，说明校正结果会用于后续病历生成。

讲解重点：

- Mock ASR 是录屏保底路线，用于稳定展示 SSE 分段追加和校正闭环。
- 中栏默认只显示时间、角色、文本和“查看原文”，工程细节放到抽屉或调试材料里。

### 2.3 真实 ASR 预热后展示

真实 ASR 不建议作为录屏第一步。先用短音频预热 FunASR 或 SenseVoice，等待模型加载完成后再开始正式录屏。

如果首次上传时中栏显示“正在识别音频，等待首段转写”，这是正常状态：

- 本地模型首次加载或下载时，后端还不能产生 `segment`。
- 前端会显示已用时和估算进度，首个分段完成后自动追加到列表。
- Docker 已把演示切片阈值设为 60 秒，1 分钟以上样例更容易出现 partial segment。

讲解重点：

- FunASR 和 SenseVoice 是真实 ASR 路线，但首次模型加载慢不代表系统卡死。
- 模型缓存保存在 `data/asr_model_cache/`，不要删除缓存目录。

### 2.4 长音频只展示证据，不现场强跑

打开以下材料说明长音频测试已做过：

- `data/asr_eval/reports/v0_5_9_chunked_long_audio/`
- `docs/final_report/images/v0_6_7_frontend_acceptance/`
- `docs/能力证据追踪矩阵.md`
- `docs/版本演进记录.md`

讲解重点：

- 16/30 分钟样本已经验证过。
- 长音频通过切片转写规避整文件失败。
- 现场不跑 30 分钟，是为了保证审核节奏，不是回避测试。

## 3. 卡顿时的处理顺序

| 现象 | 处理 | 现场话术 |
| --- | --- | --- |
| 转写区 0 段但有“等待首段” | 等待模型加载，或切到 Mock ASR | 首次加载本地模型时还不会产出 segment，系统仍在处理 |
| FunASR 长时间无结果 | 切换 Mock ASR 展示流程 | ASR 引擎可替换，病历 Agent 主流程不依赖单一模型 |
| 真实 ASR 报错 | 打开日志和评测报告 | 失败会被记录，长音频通过切片和 fallback 处理 |
| 页面仍显示旧 UI | 强制刷新浏览器并确认 Docker 已重建 | 前端文件打包在镜像内，必须 build 后才能加载 |

## 4. 不要在现场做的事

- 不上传真实患者音频或真实病历。
- 不展示 API Key、Token、模型权重路径或隐私文件。
- 不现场删除 `data/docker_runtime/` 或 `data/asr_model_cache/`。
- 不现场跑 30 分钟长音频。
- 不把 Mock ASR 说成真实模型效果。

## 5. 结束总结

建议收束句：
