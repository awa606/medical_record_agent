# v0.6.3 医生端演示脚本

本文用于课程展示和阶段验收，默认演示路线以稳定闭环为主，不把 Qwen3-ASR 作为现场默认模型。

## 演示目标

展示系统能完成：

```text
问诊输入 -> 实时转写 -> 医生/患者角色校正 -> 病历生成 -> 医生审核 -> 导出
```

## 推荐演示路线

1. 打开医生端：

   ```text
   http://127.0.0.1:8000/static/doctor.html
   ```

2. 说明顶部 6 步流程条和 `下一步` 面板。

3. 先走文本导入：
   - 点击“文本导入”。
   - 使用默认发热问诊文本。
   - 点击“确认生成”。
   - 展示字段区、候选诊断、安全校验和医生审核边界。

4. 再走音频保底路线：
   - 点击“上传音频”。
   - 选择短 MP3/WAV。
   - 选择 `Mock ASR`。
   - 展示 SSE 分段转写、角色下拉、文本编辑和保存角色校正。

5. 展示真实中文 ASR 路线：
   - 选择 `SenseVoice Small` 或 `FunASR`。
   - 上传中文问诊音频。
   - 展示转写结果和角色校正入口。
   - 如果模型下载或环境异常，说明系统会显示错误并可切换 fallback。

6. 展示长音频能力：
   - 使用 16/30 分钟稳定性样本。
   - 展示转写区的切片指标：当前切片、总切片、进度、失败原因和重试提示。

7. 完成导出闭环：
   - 点击“用校正文本生成病历”。
   - 点击“保存草稿”。
   - 点击“确认字段”。
   - 点击“确认导出”。
   - 展示导出的 Markdown/Word 路径。

## 演示时强调

- AI 只生成病历草稿和候选诊断，最终审核权在医生。
- 医学知识库规则是课程模拟知识库，不能作为真实临床诊断依据。
- Qwen3-ASR 已有评测记录，但当前普通医院 PC 交付路线优先使用 SenseVoice/FunASR。
- Mock ASR 是现场兜底，保证流程演示不中断，不代表真实模型效果。

## 演示失败处理

| 问题 | 处理 |
| --- | --- |
| 真实 ASR 初始化慢 | 切换 `Mock ASR` 完成主流程 |
| 长音频失败 | 展示失败原因和重试提示，再说明切片策略 |
| 角色识别不准 | 手动校正角色，强调医生审核 |
| 导出按钮不可用 | 先执行“确认字段” |
| 页面无法访问 | 检查本地服务是否启动 |
