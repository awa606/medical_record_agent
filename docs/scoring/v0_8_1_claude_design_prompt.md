# Claude Design PPT 提示词：Medical Record Agent 周评审

请根据以下内容制作一份 13 页中文周评审 PPT，风格为“医疗信息系统工程评审”，不要做成营销风格。整体视觉参考医院工作台：白底、浅灰分区、蓝色主色、绿色通过、红色风险、黄色提示。页面要清晰、克制、专业，适合老师评审。

## 基本信息

- 项目名称：Medical Record Agent：AI 生成式电子病历辅助系统
- 版本：v0.8.1
- 展示日期：2026-07-10
- 场景：中文医患问诊，音频/文本输入到电子病历草稿
- 边界：课程工程原型，辅助医生，不替代医生诊断

## 必须包含的页面

1. 标题页
   标题：Medical Record Agent
   副标题：中文医患问诊电子病历辅助系统 · 第 1 周工程进展评审

2. 项目目标
   说明输入、转写、角色校正、病历生成、医生审核、导出闭环。

3. 本周计划任务
   用表格展示 ASR 评测、中文样本、长音频、切片、Docker、版本记录。

4. 系统架构
   用流程图展示：医生端 -> FastAPI -> ASR/SSE -> Orchestrator -> 知识库 -> SQLite/导出。

5. 医生端主流程
   用 6 步流程条：输入、实时转写、角色校正、生成病历、医生审核、导出。

6. v0.8.1 新增实时病历预览
   重点展示 partial 转写触发左侧病历预览和右侧 AI 辅助预览。明确标注“实时预览，需医生确认”。

7. ASR 模型评测
   用对比表展示 FunASR、SenseVoice、Qwen3-ASR 的定位。解释 CER、RTF、RSS。

8. 长音频稳定性
   展示 16/30 分钟测试、5 分钟切片策略、FunASR fallback、SenseVoice 切片修复。

9. Docker 和访问方式
   展示 `docker compose up -d --build`、`http://127.0.0.1:2601/static/doctor.html`、局域网访问。

10. 第一周评分表对应
    表格四行：技术打通 8、系统架构 6、接口定义 3、过程记录 3。

11. 演示路线
    文本生成、Mock ASR、真实 ASR、长音频报告、审核导出。

12. 风险边界
    课程模拟知识库、不是临床自动诊断、角色分离需后续 diarization、医院 PC 未实机复测。

13. 下一步计划
    v0.8.2、v0.8.3、v0.9、v1.0 的路线。
