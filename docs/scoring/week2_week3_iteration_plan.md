# 第二周/第三周迭代计划与第一周效果评估

更新日期：2026-07-11  
当前版本口径：`v0.8.8`  
目标定位：课程/答辩级可交付 POC，不是医院真实上线系统。

## 第一周效果评估

第一周迭代效果总体达标，项目已经从“能跑通的 POC”推进到“可演示、可评测、可冻结材料”的阶段。当前已具备文本生成、音频上传、SSE 转写、医生端三栏工作台、播放器 Range、CAM++ 说话人校准、角色统一校正、证据回放、两说话人 RTTM 评测和 Docker 2601 前端验收。

主要证据：

- `docs/doctor_workbench_acceptance_v1_0.md`
- `data/asr_eval/reports/v0_8_8_diarization/diarization_summary.md`
- `docs/final_report/images/v1_0_frontend_acceptance/`
- `docs/scoring/v1_0_final_freeze_checklist.md`
- `logs/daily/2026-07-11.md`

短板也很明确：三说话人样本未补，SSE 断连恢复未最终验收，FunASR 首次冷启动会拖慢现场节奏，普通医院 PC 实机数据不足，真实临床产品所需的权限、审计、脱敏、合规和 HIS/EMR 对接尚未进入产品级实现。

离课程终答辩可交付已经很近，主要还差 1-2 天的演示视频、最终人工检查和提交包清理。离真实可上线产品仍有明显差距，至少还需要 2-3 周工程加固；如按医院真实临床上线标准，还需要合规、隐私、安全、临床验证和系统集成工作。

## 第二周计划：稳定性与评测补强

目标：把当前 v0.8.8 从“可演示”推进到“可稳定验收”。

| 任务 | 做法 | 验收标准 |
| --- | --- | --- |
| 三说话人评测 | 采集或整理真实三说话人课程样本，完成人工 RTTM，不合成伪样本 | 产出真实 RTTM、单样本 JSON、summary.md/json；无样本时继续标 `pending_sample` |
| SSE 断连恢复 | 验证 EventSource 重连、Last-Event-ID、重复 segment 去重和恢复后继续生成病历 | 至少有自动化或手工验收记录，失败进入 debug log |
| FunASR 冷启动加固 | 增加答辩前预热脚本或 runbook 命令，前端保持“模型加载/预热中”提示 | 预热路线可复现，现场不会被误判为前端卡死 |
| 稳定性测试 | 覆盖短音频、5 分钟、16 分钟、30 分钟样本 | 记录耗时、RTF、内存、失败原因和 fallback |
| 普通医院 PC 复测 | 记录 CPU、内存、磁盘、GPU、FunASR 首次/预热后耗时 | 形成最低配置建议 |
| Bug List 闭环 | 所有失败进入 `logs/debug/` | 明确区分前端、模型依赖、数据质量和性能问题 |

第二周验收标准：三说话人状态不再停留在口头待补；SSE 断连至少有验收记录；FunASR 现场预热路线明确；稳定性报告和 PC 复测报告可引用。

## 第三周计划：产品化交付加固

目标：把系统从“课程演示包”推进到“可交接 POC 产品包”。

| 任务 | 做法 | 验收标准 |
| --- | --- | --- |
| 医生端体验收敛 | 减少调试痕迹，保留输入、转写、校正、病历、证据、审核和导出路径 | 医生模式可独立演示，调试信息只在调试模式出现 |
| 产品级边界补齐 | 明确不接真实患者数据、AI 不自动诊断、导出前医生确认、Mock/真实 ASR 展示口径 | README、报告、runbook、表单口径一致 |
| 交付包结构整理 | 对齐 README、部署说明、runbook、验收文档、评测报告、截图、日报、debug log、表单和最终报告 | 交付目录可按冻结清单检查 |
| 提交前检查 | 排除 `.venv`、模型缓存、Docker runtime、真实音频、API Key、大文件和原始手册 | `git status --short` 无禁止提交项 |
| 答辩资产准备 | 准备录屏视频、PPT、讲稿、备用路线、老师追问答复和证据索引 | 现场可按 8-10 分钟稳定讲完 |
| 全量回归 | 运行 pytest、JS 检查、脚本编译、Docker build、health、前端截图验收、表单刷新、DOCX 导出 | 所有结果写入日报或验收文档 |

第三周验收标准：别人按 README 和 runbook 能启动系统、复现主流程、查看评测证据，并理解哪些能力已完成、哪些边界未完成。

## 优先级

如果时间不足，优先级顺序固定为：

1. 主流程稳定。
2. 三说话人真实边界。
3. SSE 断连恢复。
4. 普通医院 PC 复测。
5. 最终演示视频。
