# 本周评审展示方案：v1.3.x 产品化收敛阶段

## 当前阶段

当前项目处于 `v1.3.x 产品化收敛阶段`，不是医院正式部署版。系统已经形成课程工程闭环：输入音频/文本、ASR 转写、说话人/角色处理、病历草稿生成、候选诊断、治疗方案、证据追溯、医生确认、导出和 GitHub 版本管理。

本周重点不是继续堆新功能，而是回应老师反馈：

- 医生不一定理解模型选择，因此默认界面不强调模型。
- 医生端文字信息过多，因此默认只显示关键结果。
- 角色自动识别不能保证 100%，因此低置信度不显示为确定医生/患者。
- 前端布局仍需收敛，因此将细节放入详情和设置，不放在默认主界面。

## 5-8 分钟展示路线

1. GitHub 版本记录，30 秒
   - 打开 GitHub Releases，展示 `v1.3.2`、`v1.3.3`、`v1.3.4`。
   - 说明每个版本都有 Issue、提交、tag、Release 和测试记录。

2. 医生端入口，1 分钟
   - 打开医生端页面。
   - 展示右上角 `开始生成`。
   - 说明医生不需要理解模型参数，只选择上传音频、粘贴文本或 Mock 演示。

3. 文本生成病历，2 分钟
   - 使用粘贴文本生成一条病历。
   - 展示左侧病历草稿字段：主诉、现病史、既往史、过敏史、查体、初步诊断、处理建议。
   - 展示右侧候选诊断、治疗方案、诊断证据。
   - 强调所有诊断和治疗建议都是候选，必须医生确认。

4. ASR 与角色安全，1-2 分钟
   - 展示转写区：系统默认自动做说话人和角色分类。
   - 说明医生不需要逐句分类；低置信度由后台质量门禁兜底。
   - 说明后续路线是医生声纹注册、全局角色映射、本地 LLM 整位 speaker 判断。

5. 详情弹窗，1 分钟
   - 展示字段详情、证据、质量原因。
   - 说明默认界面减少文字，细节按需展开。

6. 风险边界和下周计划，1 分钟
   - 说明当前是课程工程原型，不声明临床可用。
   - 医院 PC 和边缘端真实部署暂不做本周目标，需等设备和合规条件明确。
   - 下周继续优化真实音频样本、角色映射质量和字段抽取质量。

## 回应老师反馈

### 1. 模型选择医生不一定懂

已调整：医生默认界面不再突出 ASR 引擎、切片、预热、质量门禁等工程细节。右上角入口变成 `开始生成`，让医生只选择输入方式。

### 2. 医生端文字太多

已调整：默认界面保留关键字段、候选诊断、治疗方案和诊断证据；详细证据、质量报告和错误原因进入详情弹窗。

### 3. 角色自动区分不可靠

已调整：默认由程序自动分类，不要求医生逐句分类。低置信度不直接写入正式病历，而是通过质量门禁、全局映射或后台提示兜底。

### 4. 前端布局不够简洁

当前状态：已经完成入口降噪和信息层级收敛，但整体视觉仍是课程工程产品形态，不是最终医院产品。后续只做遮挡、错位、过度信息等小修，不再大改三栏结构。

## 现场兜底

- 真实 ASR 加载慢：使用 Mock 演示展示完整流程。
- 角色识别不稳：说明系统默认自动分类，但低置信度会进入质量门禁，不要求医生逐句分类。
- Docker 端口冲突：使用 `scripts/check_docker_port.py` 自动选择 2600-2699 的可用端口。
- 前端被指出不够美观：承认当前是课程工程产品形态，下一步只做关键路径降噪和可用性小修。
