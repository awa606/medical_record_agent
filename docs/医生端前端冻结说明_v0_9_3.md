# 医生端前端冻结说明：v0.9.3

## 目标

v0.9.3 是医生端前端的最后一次大范围收敛。默认医生界面只保留评审和医生演示需要的一眼可读信息：病历草稿摘要、对话转写摘要、AI 辅助摘要和操作区。完整字段、证据、质量原因、ASR 细节和调试项统一进入详情抽屉或显示设置。

## 本次前端调整

- 顶部操作区收敛为 `首页`、`输入方式`、`显示设置`。
- `调试模式`、`截图模式`、`关怀版`、模型预热状态等低频项移入 `显示设置`。
- `下一步` 面板只保留当前动作和主按钮，不再显示长解释。
- 左侧 `病历草稿` 固定展示主诉、现病史、既往史、过敏史、查体、初步诊断、处理建议。
- 病历字段卡默认只显示字段名、核心摘要、一个状态点和 `详情`。
- 现病史、查体、处理建议给更大空间；既往史、过敏史更紧凑。
- 中间 `对话转写` 默认只显示时间、角色、正文和标题旁进度。
- 右侧 `AI辅助` 默认只保留候选诊断、治疗方案、诊断证据。
- 病历质量摘要改为右侧顶部一行状态，详细质量报告进入详情。
- `确认导出` 不可用时点击会打开阻断原因详情。

## 冻结规则

v0.9.3 完成后，前端进入 `frontend-freeze` 状态。后续默认只允许以下小修：

- 修复遮挡、重叠、不可点击、横向溢出。
- 修正文案错误、乱码、状态显示错误。
- 后端新增字段的最小展示适配。
- 不再重做三栏结构、颜色体系、按钮体系和整体交互模式。

## 后续主线

- `v0.9.4`：候选诊断、治疗建议和补问建议质量优化。
- `v0.9.5`：最终演示数据、稳定性和导出体验。
- `v1.0`：交付冻结、报告、GitHub Release 和演示材料。

## 验收方式

1. 打开 `http://127.0.0.1:2626/static/doctor.html`。
2. 空态下确认三栏简洁，没有调试信息和大段说明。
3. 通过 `输入方式 -> 文本生成` 生成病历。
4. 检查左侧字段摘要、右侧 AI 三块和质量状态行。
5. 点击任意 `详情`，确认能查看完整内容、证据和质量原因。
6. 未确认字段时点击 `确认导出`，确认弹出阻断原因。
7. 通过 Mock ASR 验证上传、转写、角色校正、生成病历流程不受影响。
