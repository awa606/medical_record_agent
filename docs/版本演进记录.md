## 当前状态

当前代码已推进到 `v0.8.12`。项目已经形成从音频上传、原生流式转写、真实进度、音频核对、说话人校准、整位说话人角色映射、实时病历预览，到正式生成、医生审核和导出的闭环。本轮补齐了三项工程稳定性证据：第一，使用 AliMeeting Eval 公开中文会议数据生成真实多说话人 RTTM 样本；第二，用脚本验证 500+ SSE 事件断连恢复不重复、不丢失；第三，FunASR 从手动预热改为服务启动后后台预热，并在医生端展示“模型准备中/已就绪/失败”状态。pyannote/3D-Speaker 实测和医院 PC 实机复测仍为待补项，不输出伪成绩。

## 版本管理规则

- 每个功能、修复、研究或重构任务必须先有 GitHub Issue。
- 每天至少维护一个 `/logs/daily/YYYY-MM-DD.md` 工作日志。
- Bug 修复必须在 `/logs/debug/` 下记录问题、根因、修复和验证。
- 每个里程碑必须在 `versions/` 下维护目录和 README。
- 版本目录 `versions/` 只归档关键阶段材料和证据索引，不存放大体积数据、模型权重或隐私数据。
