# Medical Record Agent 版本演进记录

本文记录 Medical Record Agent 从课程原型到研究级 AI 工程系统的标准版本线。版本线以能力里程碑为准，每个里程碑必须能追溯到 Issue、日志和验证结果。

## 版本时间线

| 版本 | 阶段 | 说明 | 归档目录 |
| --- | --- | --- |
| v0.1 | 基础 ASR 流程 | 建立音频上传、ASR 转写、ASRResult 结构和音频到病历入口。 | `versions/v0.1_basic_asr_pipeline/` |
| v0.2 | SSE 实时流式转写 | 建立任务状态、步骤记录和 SSE 事件流，支持前端实时追踪。 | `versions/v0.2_sse_streaming/` |
| v0.2.1 | ASR 会话级 SSE | 新增 MP3/WAV ASR 会话接口，支持前端中间转写栏按 segment 实时显示。 | `versions/v0.2_sse_streaming/` |
| v0.3 | 医生/患者角色校正 | 建立医生/患者角色分离、ASR 角色策略、逐段人工校正和校正后病历生成。 | `versions/v0.3_role_separation/` |
| v0.4 | 医学知识推理 | 建立病历字段抽取、草稿生成、安全校验、候选诊断和医生审核边界。 | `versions/v0.4_medical_reasoning/` |
| v0.4.1 | 医学知识库关联规则增强 | 基于 `data/output/kb/` 的症状、证候和规则库，输出候选诊断依据、建议检查、用药提示、风险提醒和建议补问。 | `versions/v0.4_medical_reasoning/` |
| v0.4.2 | 知识库可信性与演示闭环 | 增加规则来源、验证样例、验证报告和前端规则解释展示，说明当前知识库只用于课程模拟和医生复核。 | `versions/v0.4_medical_reasoning/` |
| v0.5.0 | 本地模型评测基线 | 建立硬件配置采集、mock ASR 基线、评测汇总报告和边缘端配置建议初稿，不强制下载真实模型。 | `versions/v1.0_deployable_system/` |
| v0.5.1 | 本地 ASR 多引擎评测运行器 | 新增统一评测入口，按引擎记录 measured/skipped/failed 状态；当前 mock 已测，FunASR/Qwen3 因依赖缺失被跳过。 | `versions/v1.0_deployable_system/` |
| v0.5.2 | 本地多模型 ASR 实测 | 在 `.venv-asr` 中完成 mock、FunASR、SenseVoice、Whisper、Qwen3 同样本运行；FunASR/SenseVoice 已实测，Whisper/Qwen3 记录明确阻塞。 | `versions/v1.0_deployable_system/` |
| v0.5.3 | 非医疗公开音频 ASR 冒烟测试 | 增加公开非医疗样本准备、runner smoke 模式、项目本地便携 ffmpeg、Whisper 复测和 Qwen-ASR 环境检查。 | `versions/v1.0_deployable_system/` |
| v0.5.4 | 中文优先 ASR 评测与 Qwen 复测 | 明确中文医患样本为主评测、英文公开样本仅为可选 smoke；建立普通医院 Windows PC 配置基线，并进入 Qwen-ASR Python 3.12 隔离环境复测。 | `versions/v1.0_deployable_system/` |
| v1.0 | 可部署交付系统 | 建立可本地运行、可选本地模型、日志、版本、Issue 和 PR 工作流的可交接系统。 | `versions/v1.0_deployable_system/` |

## 当前状态

当前代码已覆盖 `v0.1` 到 `v0.5.4` 的主要工程能力，并在 `v0.2.1` 增加 ASR 会话级 SSE 文件流转写，在 `v0.3` 增加医生/患者角色校正闭环，在 `v0.4.1` 增加医学知识库关联规则解释和建议字段，在 `v0.4.2` 增加知识库来源说明、规则验证报告和前端解释展示，在 `v0.5.0` 建立本地模型评测基线，在 `v0.5.1` 建立本地 ASR 多引擎评测运行器，在 `v0.5.2` 完成 FunASR/SenseVoice 本机 CPU-only 实测并记录 Whisper/Qwen3 阻塞，在 `v0.5.3` 通过便携 ffmpeg 解阻塞 Whisper 并完成公开非医疗 smoke 评测，在 `v0.5.4` 明确中文优先评测口径、普通医院 Windows PC 配置基线和 Qwen-ASR Python 3.12 复测路径。下一阶段重点是完成 Qwen 复测证据、医院 PC 实机采集和 `v0.6` 前端产品化补强。

## 版本管理规则

- 每个功能、修复、研究或重构任务必须先有 GitHub Issue。
- 每天至少维护一个 `/logs/daily/YYYY-MM-DD.md` 工作日志。
- Bug 修复必须在 `/logs/debug/` 下记录问题、根因、修复和验证。
- 每个里程碑必须在 `versions/` 下维护目录和 README。
- 版本目录 `versions/` 只归档关键阶段材料和证据索引，不存放大体积数据、模型权重或隐私数据。
