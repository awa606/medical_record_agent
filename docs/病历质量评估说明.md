# 病历质量评估说明

## 目标

v0.9.0 开始，项目主线从 ASR 输入质量转向病历生成质量。质量评估不替代医生判断，只用于提示病历草稿中哪些字段完整、哪些证据不足、哪些内容需要补问或复核。

## 评估范围

- 核心字段完整度：主诉、现病史、既往史、过敏史、查体、初步诊断、处理建议。
- 低置信度字段：由抽取置信度、待补充状态和安全校验共同判断。
- 证据覆盖率：字段是否能追溯到转写片段、文本证据或规则依据。
- 候选诊断状态：候选诊断必须保持“待医生确认”，不能写成自动确诊。
- 治疗建议安全边界：处理建议只作为医生复核提示，不自动处方。
- 是否可进入医生审核：用于提示下一步，但不绕过医生确认和导出校验。

## 接口变化

- `POST /api/records/preview` 增加 `quality_preview`。
- 正式病历任务的 `result_json` 增加 `quality_report`。
- 不新增数据库字段，不改变审核和导出接口。

## 前端展示

- 左侧病历字段增加质量标签：质量可用、需补充、低置信度、证据不足。
- 右侧 AI 区增加“病历质量摘要”，显示完整度、证据覆盖、低置信度项和下一步建议。
- 所有预览内容仍标记为“需医生确认”，未审核前不能导出。

## 安全边界

当前系统仍是课程工程原型。质量评估只能说明工程链路是否完整、证据是否可追踪，不能证明临床诊断正确性。
