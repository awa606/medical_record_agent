# 知识库规则可信性说明

本文说明 Medical Record Agent 当前知识库规则的来源、验证方法和后续扩展边界。当前规则用于课程工程演示和医生复核提示，不声明具备真实临床诊断有效性。

## 当前规则来源

当前 `v0.4.2` 使用的是课程模拟知识库，主要文件位于：

- `data/output/kb/symptom.json`：症状、标准化表达和同义词。
- `data/output/kb/syndrome.json`：证候、舌脉和治法方向。
- `data/output/kb/rule_base.json`：阳性症状、反向症状和规则权重。
- `data/output/kb/rule_sources.json`：规则来源、审核状态和使用限制。

当前所有规则来源类型均为 `course_mock`，审核状态为 `needs_medical_review`。系统输出只能作为“候选/待医生确认”的辅助提示。

## 规则判断方式

系统不会直接给最终诊断。规则服务按以下流程生成候选提示：

1. 从问诊文本中匹配症状名称、标准化表达和同义词。
2. 用 `rule_base.json` 中的 `positive_symptom_ids` 计算阳性症状命中。
3. 用 `negative_symptom_ids` 降低反向线索对应规则的置信度。
4. 生成 `CandidateDiagnosis`，包含 `rule_id`、`confidence`、`reason`、`suggested_checks`、`medication_notes`、`risk_warnings` 和 `follow_up_questions`。
5. 前端显示规则依据，但导出前仍要求医生审核。

## 怎么判断规则是否正确

目前只能判断两类“正确”：

- 工程正确：给定测试样例，规则是否命中预期 `rule_id`，是否避免误触发，是否保持候选状态。
- 审核正确：规则来源是否清楚，是否标记人工审核状态，是否明确不替代医生。

不能仅凭当前测试证明医学正确。真实医学正确性需要医学教材、指南、专家审核和临床数据验证。

## 验证方法

运行：

```powershell
$env:PYTHONPATH = (Get-Location).Path
python scripts/validate_kb_rules.py
pytest -q tests/test_kb_rule_validation.py tests/test_knowledge_rules.py
```

验证输入：

- `data/output/kb/rule_validation_cases.json`

验证输出：

- `data/output/kb/rule_validation_report.json`
- `data/output/kb/rule_validation_report.md`

验证样例覆盖风寒、风热、暑湿、气虚、非感冒负例和风险提醒样例。

## 后续导入教材、指南或书籍知识的流程

后续可以加入教材、指南或书籍内容，但不能直接把大段版权原文提交到仓库。建议流程：

1. 选定来源：教材、指南、老师给定材料或公开授权资料。
2. 提取结构化知识点：疾病、症状、鉴别点、危险信号、建议检查、补问问题。
3. 记录来源元数据：书名、章节、页码或链接、整理人、整理日期、审核状态。
4. 写入结构化规则：只保存规则字段和短证据说明，不保存大段原文。
5. 增加验证样例：包含正例、负例和安全边界样例。
6. 人工审核通过后，将 `review_status` 从 `needs_medical_review` 改为更高可信状态。

## 当前边界

- 当前知识库是课程模拟知识库，不用于真实诊疗。
- 系统不输出最终诊断，不生成处方。
- 风险提醒用于提示医生复核，不等同于分诊或急诊决策。
- 规则置信度是工程规则分数，不是医学概率。
