# 说话人角色区分测试与模型提升计划

## 当前结论

系统不能承诺“自动 100% 区分医生/患者”。当前可承诺的是产品行为：

- 低置信度自动结果不直接显示为“医生/患者”。
- 默认降级为“说话人 A/B/C”，由医生做一次全局映射确认。
- 病历生成只使用稳定转写段，不使用临时 streaming 文本或混合段。
- 每轮模型改动必须通过固定样本和指标门禁。

## 固定评测集

| 样本 | 用途 | 当前定位 |
| --- | --- | --- |
| `fever_01` | 中文医患双人问诊 | 主评测样本 |
| `chest_pain_01` | 中文医患双人问诊 | 主评测样本 |
| `snakebite_01` | 单人朗读/课程样本 | 防止伪造医患双人 |
| `three_speaker_alimeeting_01` | 中文三说话人公开样本 | 多说话人分离评测，不代表医疗场景 |
| 医患+家属模拟样本 | 后续补充 | 用于三角色产品验收 |

## 质量指标

| 指标 | 含义 | 用途 |
| --- | --- | --- |
| DER/JER | 说话人分离错误率 | 衡量 diarization 整体效果 |
| speaker count error | 说话人数误差 | 防止把单人伪造成多人，或漏掉家属 |
| boundary F1 | 说话人切换边界准确性 | 减少医生和患者句子混在一行 |
| mixed utterance rate | 混合语句率 | 直接衡量“串话”风险 |
| role accuracy | 医生/患者/其他角色准确率 | 衡量角色映射效果 |
| manual confirmation rate | 需要人工确认比例 | 衡量自动化程度和医生负担 |
| RTF/RSS | 处理速度和内存 | 判断普通医院 PC 是否可运行 |

## v1.3.2 质量门禁

新增本地脚本：

```powershell
python scripts\validate_speaker_role_quality.py `
  --asr-result data\asr_eval\reports\sample_asr_result.json `
  --output-json data\asr_eval\reports\speaker_role_quality.json `
  --output-md data\asr_eval\reports\speaker_role_quality.md
```

门禁规则：

- `role_confidence < 0.9` 且未人工确认的临床角色，不能直接展示为医生/患者。
- 未完成角色映射时，医生端显示说话人 A/B/C。
- 单人朗读样本不能伪造成医生和患者两个人。
- 生成病历前如有 unresolved speaker，只触发一次全局映射确认。

## 提升路线

1. 先补数据和标注：RTTM、speaker-role、字段证据。
2. 优先优化 VAD + diarization 边界，减少混合语句。
3. 增加医生声纹注册，用本机声纹优先锁定医生。
4. 使用本地 LLM 只判断整位 speaker 的角色，不逐句猜测。
5. 若 FunASR CAM++ 不达标，再实测 pyannote 和 3D-Speaker。
6. ASR 大模型微调放到后续研究阶段，当前不从零训练。

## 周评审说明

对老师说明时应避免说“已经完全自动识别医生/患者”。推荐说法：

> 当前系统已经把自动识别结果纳入质量门禁。高置信度结果可直接辅助展示；低置信度结果不会误导医生，而是显示为说话人 A/B，并通过一次全局确认进入病历生成流程。
