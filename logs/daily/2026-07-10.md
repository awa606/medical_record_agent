## 上传音频准实时转写早期原型（已由 v0.8.3 取代）

### 本轮目标

- 修复医生端“上传音频后不是每秒或几秒显示转写”的体验问题。
- 将演示链路从“整段或大切片完成后吐 segment”改为“上传音频按小切片逐段吐 segment”，并联动病历实时预览。

### 已完成

- 后端 `app/api/asr_sessions.py` 新增上传音频准实时模式：
  - 默认支持 `mock`、`funasr`、`sensevoice`。
  - 新增 `ASR_SESSION_REALTIME_UPLOAD_ENABLED`、`ASR_SESSION_REALTIME_CHUNK_SECONDS`、`ASR_SESSION_REALTIME_CHUNK_MIN_SECONDS`、`ASR_SESSION_REALTIME_MAX_SECONDS`、`ASR_SESSION_REALTIME_MOCK_DELAY_SECONDS` 配置。
  - 短音频可按 3 秒小切片顺序识别，每个小切片完成后立即写入 `segment` SSE 事件。
  - `mock` 引擎按时间轴分批发送 segment，并在 Docker 演示环境中按 2.2 秒节奏显示，适合作为录屏保底。
  - 保留原长音频切片逻辑，准实时模式之外仍可使用 60/300 秒稳定切片。
- 前端 `static/doctor.js` 调整实时体验：
  - 降低病历预览触发门槛到 1 段或 10 字，最短间隔约 1.8 秒。
  - 每个 `segment` 到达时只追加当前行，并记录首段时间、最近分段时间和已转写音频秒数。
  - 中栏状态 badge 显示“转写中 · N段 · 已转写X秒”。
- Docker 2601 演示配置已更新并重建：
  - `ASR_SESSION_REALTIME_CHUNK_SECONDS=3`
  - `ASR_SESSION_REALTIME_MOCK_DELAY_SECONDS=2.2`
  - `ASR_SESSION_REALTIME_MAX_SECONDS=600`

## v0.8.3-v0.8.5 原生流式、播放器与多说话人联动

### 本轮目标

- 把 3 秒临时文件切片原型收敛为 Paraformer Streaming 原生上传音频流式识别。
- 区分播放时间、转写进度和程序处理用时，去掉虚假百分比和固定 8 秒伪时间戳。
- 增加音频核对播放器、CAM++ 多说话人校准、整位说话人角色修订。
- 让稳定转写持续驱动病历字段和 AI 辅助预览，并可从证据回放来源音频。

### 已完成

- `v0.8.3`：新增 `funasr_streaming_engine.py`，上传音频一次解码为 16 kHz PCM，按约 600 ms 帧持续识别；SSE 增加稳定段 ID、原位修订和实际进度字段。
- `v0.8.4`：新增 Range 媒体接口和中栏播放器，支持播放、暂停、拖动、音量、倍速、尽快识别/跟随播放；离线全局校准启用 FSMN-VAD、标点和 CAM++。
- `v0.8.4`：`speaker_id` 与 `role` 分开保存，支持把一位说话人的全部发言统一改为医生、患者、其他或待确认。
- `v0.8.5`：实时预览请求最多每 2 秒一次，新请求取消旧请求；结构化字段返回来源 `segment_id`，证据按钮可定位播放。
- 修复 FunASR 句级时间戳毫秒值被误判为秒的问题，记录见 `logs/debug/2026-07-10_funasr_timestamp_unit_mismatch.md`。
- 创建 GitHub Issues：[#7](https://github.com/awa606/medical_record_agent/issues/7)、[#8](https://github.com/awa606/medical_record_agent/issues/8)、[#9](https://github.com/awa606/medical_record_agent/issues/9)。

### 实际验证

- Docker `2601` 已完成真实中文短音频上传冒烟：事件顺序包含 `segment`、`segment_update`、`diarization_progress`、`reconciliation_completed` 和 `completed`。
- `GET /api/audio/{audio_id}/media` 的 Range 请求返回 `206 Partial Content`，播放器可拖动。

- 单元测试覆盖流式分帧、进度单调、稳定段 ID、CAM++ 映射、全局角色修订、媒体 Range 和证据来源。

### 前端展示调整

- 中栏底部新增音频播放器和两种显示节奏。
- 转写行按真实时间戳显示；缺失时间戳时显示 `--:--`。
- 临时转写行可在全局校准后原位更新，不重复追加。
- 点击转写行或诊断证据可以跳转并播放对应音频。
- 左右栏实时预览统一标记“实时预览，需医生确认”。

### 前端怎么测试

1. 打开 `http://127.0.0.1:2601/static/doctor.html`。
2. 选择 FunASR，上传中文短 WAV 或 MP3。
3. 确认模型加载阶段不显示百分比，识别阶段文字和真实进度持续变化。
4. 测试播放/暂停、拖动、音量和倍速。
5. 点击一条转写，确认播放器跳到对应位置。
6. 切换“尽快识别/跟随播放”。
7. 等待 CAM++ 校准，确认转写行可原位更新；修改整位说话人的角色并保存。
8. 确认左侧病历预览和右侧 AI 预览会更新，证据按钮可回放来源。

### 当前边界

- 本轮只处理已上传音频，不接浏览器麦克风。
- CAM++ 只能聚类声学身份，不能保证自动判断医生、患者或家属。
- 单人朗读脚本不会被伪造成两个真实声纹，“待确认”仍是必要安全状态。
