# 每日任务记录：三说话人评测闭环与角色映射稳定化

## 今日目标

- 补齐 AliMeeting 三/多人样本到 ASRResult 的真实转写闭环。
- 将 ASRResult 接入 diarization RTTM 评测，输出真实 measured/failed 结果。
- 修复混合语句后处理：有 diarization turn 时按说话人边界拆分 ASR segment。

## 真实评测结果

样本：`three_speaker_alimeeting_01.wav`  
来源：AliMeeting Eval，CC BY-SA 4.0。  
场景：公开会议音频，仅用于多说话人分离工程评测，不代表医疗问诊效果。

FunASR 实测：

- `status = measured`
- `audio_duration_seconds = 240.0`
- `elapsed_seconds = 26.152`
- `rtf = 0.109`
- `rss_peak_mb = 4237.45`
- `raw_segments = 1`
- `raw_speaker_count = 1`
- `diarization_turns = 0`

Diarization RTTM 评测：

- `status = failed`
- `reason = engine returned no diarization turns for a non-empty RTTM reference`

结论：当前 FunASR ASRResult 对该 AliMeeting 三/多人会议样本没有返回可对齐的说话人时间边界。后处理逻辑已经具备，但还需要接入 pyannote、3D-Speaker 或 FunASR 专门 diarization 管线，才能真正降低混合语句率。
