## v0.9.2 目标：病历字段抽取、质量规则和证据定位优化

- 将迭代主线从 ASR 输入质量切回病历生成质量。
- GitHub Issue：[#22](https://github.com/awa606/medical_record_agent/issues/22)。
- 优化字段抽取规则：主诉保留“症状 + 持续时间”，查体、过敏史和既往史未明确提及时保持缺失，不编造“无”或“正常”。
- 细化字段质量状态：`complete`、`missing`、`low_confidence`、`evidence_missing`、`needs_doctor_review`。
- `SourceSpan` 增加 `segment_id`，预览和正式结果的证据可映射到稳定转写段。
- 医生端只补充字段质量标签和病历质量摘要，不大改页面布局。

## v0.9.2 前端验收方式

1. 打开 `http://127.0.0.1:2626/static/doctor.html`。
2. 通过 `输入方式 -> 文本生成` 导入发热或蛇咬伤问诊文本。
3. 生成病历后检查左侧字段标签：查体未提及时应显示需补充，不应显示质量可用。
4. 检查右侧病历质量摘要：可看到完整度、证据覆盖、低置信度项、证据不足项和下一步建议。
5. 点击证据入口，应定位到对应转写段；有音频时应能跳转到对应时间。
6. 未医生确认前仍不能导出。

## v0.9.3 目标：医生端前端最终收敛与冻结

- GitHub Issue：[#23](https://github.com/awa606/medical_record_agent/issues/23)。
- 本轮只做最后一次前端展示收敛，不新增后端 API。
- 默认医生界面减少信息量：摘要留在主界面，证据、质量原因、ASR 细节和显示选项进入详情抽屉或显示设置。

- 左侧病历草稿字段只显示字段名、摘要、状态点和详情；现病史、查体、处理建议权重更高。
- 中间转写区默认只显示时间、角色、正文和标题旁状态。
- 右侧 AI 辅助只保留候选诊断、治疗方案、诊断证据，病历质量摘要改为一行状态。
