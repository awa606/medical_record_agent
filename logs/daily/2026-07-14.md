## v0.9.7 追加记录：Docker 端口预检与部署体验复测

### 本阶段完成
- 扩展 `scripts/check_docker_port.py`：保留 `--port` 单端口检查，并新增 `--start/--end` 范围扫描和 `--format text|env|json` 输出。
- 更新 `README.md` 和 `docs/docker_local_deploy.md`：不再把 `2601` 或 `2626` 写成固定可用端口，改为启动前扫描 `2600-2699` 并设置 `MRA_HOST_PORT`。
- 创建 GitHub Issue `#28` 跟踪本阶段部署体验复测。

### 验证方式
- `python scripts/check_docker_port.py --start 2600 --end 2699 --format env`：输出可用于 PowerShell 的 `MRA_HOST_PORT=<port>`。
- `pytest -q tests/test_check_docker_port.py`：验证端口解析、排除范围判断和范围扫描推荐逻辑。
- Docker 使用实际可用端口 `2644` 完成健康检查和医生端页面访问。

### 结论
- 当前机器上 `2601/2626` 都可能被 Windows TCP 保留端口范围影响，因此后续演示不再承诺固定端口，统一按脚本推荐的实际端口启动。
